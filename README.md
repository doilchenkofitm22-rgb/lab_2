# Опис структури.

* **policy/** - містить політику інформаційної безпеки та примітки до мапування контролів.  
  * `01_Security_Policy.md` - повна політика інформаційної безпеки із визначенням принципів, ролей, процедур контролю доступу, журналювання, управління інцидентами, ризиками, постачальниками тощо.  
  * `AnnexA_Mapping_Notes.md` - пояснення до мапування розділів політики на контролі ISO/IEC 27001 Annex A та функції NIST CSF.
* **governance/** - документи щодо підтримки та оновлення політики.  
  * `Policy_Update_Method.md` - методика регулярного перегляду та оновлення політики, опис тригерів, відповідальних осіб та процесу затвердження.
* **audit/** - документи, що описують внутрішній аудит та результати оцінки.  
  * `Internal_Audit_Procedure.md` - процедура проведення внутрішнього аудиту відповідності політиці.  
  * `Audit_Report.md` - звіт аудиту із висновками та рекомендаціями, а також таблицею оцінки зрілості.
* **assessment/** - підсумкова оцінка зрілості.  
  * `Maturity_Assessment.md` - підсумок первинної оцінки зрілості системи інформаційної безпеки та рекомендації щодо підвищення рівня.

Документи доповнюють один одного: політика встановлює загальні принципи, методика визначає, як її оновлювати, процедура аудиту описує перевірку виконання, а звіт і оцінка зрілості надають фактичні результати і рекомендації для подальшого розвитку.
